# 📥 File Downloader with Telegram Bot & GitHub Actions

<div dir="rtl">

**سیستم دانلود فایل از لینک‌های عمومی و ذخیره در ریپازیتوری گیتهاب**  
با دو روش:
1. **ویرایش فایل `trigger_download.txt`** – ساده و مستقیم  
2. **ربات تلگرام** – فقط با ارسال لینک  

ربات با تکنیک **Long Polling** تقریباً همیشه آنلاین است و امکانات زیر را دارد:

- دانلود هر نوع فایل (با دنبال کردن خودکار Redirectها)
- دانلود **موازی** (چند فایل هم‌زمان)
- **فشرده‌سازی Zip** (چند فایل با هم یا با دستور `/zip`)
- مشاهده‌ی لیست فایل‌های دانلود شده (`/list`)
- بدون نیاز به سرور خارجی (همه چیز روی گیتهاب)

## 🚀 راه‌اندازی سریع

### ۱. Fork کردن ریپازیتوری
همین پروژه را Fork کنید تا یک نسخه در اکانت خود داشته باشید.

### ۲. تنظیم متغیرهای مخفی (Secrets)
به مسیر **Settings → Secrets and variables → Actions** بروید و دو راز زیر را اضافه کنید:

| نام متغیر         | توضیح |
|-------------------|-------|
| `TELEGRAM_TOKEN`   | توکن ربات تلگرام (از [BotFather](https://t.me/BotFather) بگیرید) |
| `GH_PAT`           | Personal Access Token از گیتهاب با دسترسی کامل به مخزن (اسکوپ `repo`). [نحوه ساخت](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens) |

### ۳. فعال‌سازی ورک‌فلوها
پس از Fork، ورک‌فلوها به‌طور خودکار فعال نیستند. ابتدا یک بار به‌طور دستی اجرا کنید یا یک Push کوچک انجام دهید تا فعال شوند.

### ۴. استفاده از ربات تلگرام
به ربات خود در تلگرام پیام دهید:
- **لینک(ها) را بفرستید** → ربات فایل‌ها را دانلود کرده و لینک دسترسی می‌دهد.
- اگر **چند لینک** بفرستید، همه را در یک فایل `zip` قرار می‌دهد.
- **دستور `/zip`** قبل از لینک‌ها، فشرده‌سازی را حتی برای یک فایل فعال می‌کند.
- **دستور `/list`** لیست تمام فایل‌های موجود در پوشه‌ی `downloads` را برمی‌گرداند.

### ۵. استفاده از روش فایلی
فایل `trigger_download.txt` را ویرایش کنید، لینک(های) خود را وارد کنید و Commit بزنید.  
پس از چند ثانیه، ورک‌فلو اجرا شده و فایل‌ها در مسیر `downloads/` قرار می‌گیرند. فایل محرک خودکار به حالت اولیه برمی‌گردد.

## ⚙️ جزئیات فنی

- **زبان برنامه‌نویسی**: Go (گولنگ)  
- **ربات**: هر **۱ ساعت** یک بار اجرا می‌شود و با Long Polling به‌مدت **۵۰ دقیقه** آنلاین می‌ماند.  
  به دلیل استفاده از `concurrency`، وقفه بین اجراها معمولاً کمتر از ۱۰ دقیقه است.  
- **دانلود موازی**: حداکثر ۵ دانلود هم‌زمان برای افزایش سرعت.  
- **پشتیبانی از Redirect**: تمام لینک‌ها تا ۱۰ مرحله دنبال می‌شوند.  
- **فشرده‌سازی**:  
  - روش ربات: تشخیص خودکار چند فایل یا دستور `/zip`.  
  - روش فایلی: گذاشتن `/zip` در یک خط از فایل محرک، همه‌ی فایل‌ها را تبدیل به یک `zip` می‌کند.  
- **محدودیت حجم فایل**: گیتهاب برای فایل‌های بالای ۱۰۰ مگابایت محدودیت دارد. برای فایل‌های بزرگ‌تر پروژه مناسب نیست.

## 🛠 رفع اشکال

- **ربات پاسخ نمی‌دهد**: مطمئن شوید `TELEGRAM_TOKEN` و `GH_PAT` درست هستند و ورک‌فلو فعال شده است.  
  همچنین می‌توانید از `workflow_dispatch` در اکشن `Telegram Bot` استفاده کنید تا اجرا را مجبور کنید.  
- **فایل در ریپو ظاهر نمی‌شود**: ممکن است نیاز باشد چند ثانیه صبر کنید و صفحه را refresh کنید.  
- **خطای دسترسی (۴۰۳)**: توکن گیتهاب دسترسی `repo` را ندارد.
